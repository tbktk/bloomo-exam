## 4. UserID値オブジェクト化・Null許容の検討

**👤 質問 / 依頼**

IDと論理型のデータは値オブジェクトとしないほうがよいでしょうか？ それと、今回のケースの場合にNull許容を配慮するデータはありますか？

**🤖 回答 / 実装**

良い問いです。順番に答えます。

### UserID → 値オブジェクト化を推奨

理由: `int` のままだと別のIDと混同するリスクがある。将来 `OrderID` や `TradeID` が増えたときに型安全性が効いてくる。

### Tradable（論理型）→ 値オブジェクト化しない

理由: `bool` はそれ自体が既に型安全で、バリデーションする余地もなく、ラップしてもコードが複雑になるだけ。`Stock` エンティティの `IsTradable() bool` メソッドで十分意図が表現できる。

### Null許容を配慮すべきデータ（2箇所）

- `FindByUserID` の結果 — 存在しないユーザーIDでリクエストが来た場合。`var ErrNotFound = errors.New("portfolio not found")` で対処
- `FindByTicker` の結果 — ポートフォリオに含まれる銘柄がマスタに存在しない可能性。同様に `ErrNotFound` で対処

### usecase/trade_usecase.go（主要部分）

```go
func (u *TradeUsecase) Execute(input TradeInput) (*TradeOutput, error) {
    userID, err := vo.NewUserID(input.UserID)
    if err != nil {
        return nil, fmt.Errorf("invalid user_id: %w", err)
    }
    amount, err := vo.NewAmount(input.Amount)
    if err != nil {
        return nil, fmt.Errorf("invalid amount: %w", err)
    }
    pf, err := u.portfolioRepo.FindByUserID(userID)
    if err != nil {
        return nil, fmt.Errorf("portfolio not found: %w", err)
    }
    stocks, err := u.stockRepo.FindAll()
    if err != nil {
        return nil, fmt.Errorf("failed to fetch stocks: %w", err)
    }
    orders, err := u.calculator.Calculate(amount, pf, stocks)
    if err != nil {
        return nil, fmt.Errorf("failed to calculate orders: %w", err)
    }
    return toTradeOutput(input.Amount, pf, orders), nil
}
```

### 設計上の判断ポイント

- `TradeInput/TradeOutput` を usecase で定義する理由: handler が domain の型に直接依存しないようにするため（handler → usecase → domain の一方向を守る）
- `toTradeOutput` 関数を切り出す理由: Execute のロジックとプレゼンテーション変換を分離するため
- エラーは `fmt.Errorf("%w")` でラップする理由: 呼び出し元で `errors.Is` による種別判定ができるようにするため

## 7. ドメインサービス Calculate の実装

**👤 質問 / 依頼**

はい、Calculateの実装をお願いします。

**🤖 回答 / 実装**

では実装します。

### domain/trade/service.go

```go
func (c *OrderCalculator) Calculate(
    amount vo.Amount,
    portfolio *portfolio.Portfolio,
    stocks []*stock.Stock,
) ([]*Order, error) {
    stockMap := makeStockMap(stocks)

    // 1. ポートフォリオからtradable=falseを除外
    tradableTargets := filterTradable(portfolio.TargetPortfolio, stockMap)
    if len(tradableTargets) == 0 {
        return []*Order{}, nil
    }

    // 2. 按分計算（切り捨て）・200円未満を除外
    orders := calcOrders(amount, tradableTargets, stockMap)
    return orders, nil
}

func calcOrders(amount vo.Amount, targets map[vo.Ticker]vo.Ratio,
    stockMap map[vo.Ticker]*stock.Stock) []*Order {
    orders := make([]*Order, 0, len(targets))
    for ticker, ratio := range targets {
        // 按分金額計算（整数除算=切り捨て）
        orderAmountValue := (amount.Value() * ratio.Value()) / 100
        orderAmount, err := vo.NewOrderAmount(orderAmountValue)
        if err != nil { continue }  // 200円未満は除外
        s := stockMap[ticker]
        quantity, err := vo.NewQuantity(orderAmountValue, s.Price.Value())
        if err != nil { continue }
        orders = append(orders, &Order{
            Symbol: ticker, Amount: orderAmount, Quantity: quantity,
        })
    }
    return orders
}
```

### 実装上の判断ポイント

- 按分計算の切り捨て: `(amount * ratio) / 100` を整数除算で行うことで仕様通りの切り捨て
- 再按分はしない: `tradable=false` 除外後・200円未満除外後、残った銘柄で再計算しない（テストで合意した解釈）
- errを返さない設計: 各銘柄の除外は「異常」ではなくビジネスルールとして期待される動作のため `continue` で読み飛ばす

```go
func (h *TradeHandler) Handle(w http.ResponseWriter, r *http.Request) {
    if r.Method != http.MethodPost {
        respondWithError(w, r, http.StatusMethodNotAllowed, ErrCodeMethodNotAllowed,
            "only POST method is allowed")
        return
    }
    userIDStr := r.PathValue("user_id")
    userID, err := strconv.Atoi(userIDStr)
    if err != nil {
        respondWithError(w, r, http.StatusBadRequest, ErrCodeInvalidInput,
            "invalid user_id parameter", "user_id must be a positive integer")
        return
    }
    // ... リクエストデコード・バリデーション・ユースケース実行
    if errors.Is(err, portfolio.ErrNotFound) {
        respondWithError(w, r, http.StatusNotFound, ErrCodeNotFound, ...)
        return
    }
}
```

### main.go（DIの組み立て）

```go
func main() {
    logger.Init("$HOME/.bloomo-exam/logs", logger.INFO, false)
    stockRepo     := memory.NewStockRepository()
    portfolioRepo := memory.NewPortfolioRepository()
    calculator    := &trade.OrderCalculator{}
    tradeUsecase  := usecase.NewTradeUsecase(stockRepo, portfolioRepo, calculator)
    tradeHandler  := handler.NewTradeHandler(tradeUsecase)

    mux := http.NewServeMux()
    mux.HandleFunc("POST /users/{user_id}/trades", tradeHandler.Handle)

    var h http.Handler = mux
    h = handler.RecoveryMiddleware(h)
    h = handler.LoggingMiddleware(h)
    http.ListenAndServe(":1111", h)
}
```

- `errors.Is` でドメイン層の `ErrNotFound` を検知して404を返す
